Architecture

Designed to not fail when it matters most.

Tradelab replaces monolithic, thread-bound OMS designs with a stateless, co-routine based cluster. Every component scales independently. No shared failure paths.

Traditional OMS — What breaks
What breaks under load
✗ Single ORMS instance per client group
✗ Tightly coupled exchange adapters — one failure = full outage
✗ Thread-per-client model — memory explosion at peak
✗ UI, risk, and execution in one monolithic process
✗ No horizontal scalability — vertical only, expensive
Tradelab OMS — What runs without fear
What runs without fear
✓ Stateless cluster — infinite horizontal scale on Kubernetes
✓ Shared adapter pool — NSE/BSE/MCX completely decoupled
✓ Go co-routines — lightweight and predictable under peak load
✓ Risk engine as independent microservice — fully isolated
✓ Redis state, Kubernetes-managed, multi-AZ with DR failover
